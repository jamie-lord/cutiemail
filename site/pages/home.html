A mail server,
built from the byte up.

cutiemail is the SQLite of email: a small, self-contained mail server in TypeScript you can actually read. It sends and receives real internet mail, works with Thunderbird and Apple Mail, and keeps your mailbox in plain SQLite files you own. Zero runtime dependencies, and honest about what it leaves out.

$ npm start host real email → read the code →
0 runtime deps 1,000+ tests SPF · DKIM · DMARC passing, live MIT

Two minutes, on the machine in front of you. You need Node.js 22.18 or newer (node --version to check; Debian/Ubuntu's apt version is too old, and npm ships inside Node) and git, which most machines already have. Putting it on a real server with real DNS is the deployment guide.

your new mail server
$ git clone https://github.com/jamie-lord/cutiemail
$ cd cutiemail
$ npm install # dev tooling only, nothing the server runs
$ npm start # SMTP + submission + IMAPS, up. Leave it running
a second terminal
$ node src/main.ts selftest demo # password: demo
ExperimentalWarning: SQLite is an experimental feature… (Node's notice, not ours; harmless)
✓ selftest PASSED: authenticated submission, local delivery, and IMAP read-back all work.

A demo/demo account is seeded so it works out of the box (on your machine only; it never comes to life on a public server). Point Thunderbird or Apple Mail at IMAP 127.0.0.1:5993 and SMTP 127.0.0.1:5587 (STARTTLS). The username is demo, not an email address. Accept the one-time self-signed-certificate warning, and say hi to yourself.

A whole mail server. Actually whole.

No mail libraries: the SMTP and IMAP engines, the MIME parser, and the DKIM crypto are all code in this repo.

↓

Receive

SMTP with STARTTLS. Every inbound message is authenticated (SPF, DKIM, DMARC over real DNS). Smuggling-class bytes are rejected at the door, and forged auth headers are stripped. Politely.

↑

Send

Submission over TLS, DKIM signing, MTA-STS, and a persistent SQLite retry queue that backs off for ~5 days before sending a proper bounce. Nothing is ever silently dropped.

✓

Read

IMAPS with the surface real clients actually drive (IDLE, QRESYNC, SPECIAL-USE, per-part fetch), so your phone and your desktop stay in agreement about your mailbox.

bytes, never strings: a message is a Buffer from the socket to the SQLite BLOB and back, byte-exact.

A very good pretend internet.

Building an app that sends mail? Run cutiemail as a Mailpit-style sink. Except it's a real SMTP/IMAP server, so your code exercises real protocol behaviour. hold mode guarantees nothing ever escapes the box.

$ MAIL_CONTROL_DB=:memory: \
MAIL_OUTBOUND=hold npm start
✓ ephemeral, and nothing is ever relayed

Honestly? You might want something else.

Stalwart, Maddy, Mox, and Mailcow are all good software. If you want a batteries-included groupware stack or JMAP, use them. cutiemail's bet is to be the SQLite of email. Not the embedded-library part (email needs a server: someone has to answer when the rest of the internet knocks on port 25), but everything else it's loved for:

Radical smallness you can actually read

♥One process, one language, zero runtime dependencies
♥Plain SQLite files you can query with stock sqlite3
♥Every protocol byte is code in this repo, nothing imported, nothing hidden
♥Leaving is as easy as arriving: byte-exact storage, plain .eml out

Deliberately not here

Each omission is a recorded decision with reasons, not a gap.

no POP3 no JMAP no Sieve no webmail no clustering no spam filter no multi-domain

One domain, a handful of humans, on a small box you own. That's what it serves best.

THE TEST BED IS THE STAR OF THE SHOW

Tests that are proven to catch their own lies

Every conformance check runs both ways: pass against a clean server, fail against a mutant server with exactly the defect it targets. A test never shown to fail doesn't count as coverage.

The receiver's test bed doubles as a standalone SMTP conformance suite you can point at any mail server, calibrated against Exim, mox, and aiosmtpd with zero false positives.

$ node src/cli.ts run \
--config reference-servers/exim-local.json
✓ 59 conformant · 0 false positives
~ 2 divergences: real, triaged, recorded
# aim it at any live SMTP server (setup in reference-servers/)
# exits non-zero on findings, so it drops straight into CI
!

An honest note before you trust it

cutiemail is young (v0, one maintainer) but held to a high verification bar: over a thousand tests, each proven able to fail against a deliberately broken build, with a production instance exchanging authenticated mail with Gmail daily. Today it belongs on a spare domain or a subdomain, not as the only home of mail you can't lose. There's no spam filter or third-party security review yet, and a brand-new server's IP earns its sending reputation slowly. The full list of limitations, and the cutover path for later, is in the deployment guide.
